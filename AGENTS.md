# Stock AI Lab 개발 규칙

- 이 프로젝트는 개인용 AI 주식 분석 및 모의투자 프로그램이다.
- 실제 주식 매매 기능은 구현하지 않는다.
- Python 코드에는 가능한 한 타입 힌트를 사용한다.
- 기능별로 모듈을 분리하고 하나의 파일에 기능을 몰아넣지 않는다.
- API 키, 비밀번호, 접근 토큰을 코드에 직접 작성하지 않는다.
- 민감한 정보는 .env에서 불러온다.
- .env 파일은 Git에 포함하지 않는다.
- 모든 외부 API 호출에는 타임아웃과 예외 처리를 적용한다.
- 오류를 숨기거나 빈 except 문으로 무시하지 않는다.
- 오류 로그 context에는 API 키, 인증 토큰, 비밀번호를 저장하지 않는다.
- 데이터베이스 모델 변경 시 테스트를 수정한다.
- 운영 데이터베이스를 테스트에서 사용하지 않는다.
- 오류 API는 입력값을 검증해야 한다.
- Error Center 오류 자체가 발생하면 원래 오류를 덮어쓰지 않게 한다.
- 중요한 오류는 추후 Error Center에 저장할 수 있도록 설계한다.
- 분석 당시 실제로 이용할 수 있었던 데이터만 사용한다.
- 미래 데이터가 과거 학습 데이터에 들어가지 않게 한다.
- 머신러닝 모델은 기존 모델보다 검증 성능이 좋을 때만 교체한다.
- 중요한 기능에는 pytest 테스트를 작성한다.
- 기능 수정 후 관련 테스트를 실행한다.
- 요청받지 않은 대규모 리팩터링은 하지 않는다.
- 프로젝트 폴더 밖의 파일은 수정하지 않는다.
- 작업 결과와 테스트 결과를 명확하게 보고한다.
- GPT가 제시한 ticker와 회사명을 직접 신뢰하지 않는다.
- 관심 후보는 security master 검증을 통과해야 한다.
- 종목은 국가·거래소·ticker 조합으로 식별한다.
- ambiguous 종목 후보를 자동 추천하지 않는다.
- 회사명 exact match만 믿지 않고 ticker, security key, 공식명, 한글/영문/약칭/브랜드명 alias, issuer name을 함께 고려한다.
- 비상장 기업에 가짜 ticker를 만들지 않는다.
- ETF 후보는 ETF 이름만 보지 않고 테마 태그와 가능한 경우 구성종목 노출도를 함께 고려한다.
- ETF 구성종목 데이터가 없으면 그 한계를 UI와 진단 정보에 표시한다.
- 주식과 ETF를 분리해서 저장하고 매칭한다.
- 테스트에서 실제 외부 기준정보 API를 호출하지 않는다.
- 미국 상장 종목 목록은 Nasdaq Trader Symbol Directory를 우선한다.
- SEC 자료는 CIK와 회사명 보강에 사용한다.
- SEC에 없다는 이유로 종목이나 ETF를 삭제하지 않는다.
- Test Issue와 비추천 증권을 일반 주식으로 분류하지 않는다.
- 부분·빈 응답으로 기존 종목을 대량 비활성화하지 않는다.
- 외부 동기화 테스트는 fixture와 mock으로 수행한다.

## 뉴스 품질 규칙

- 중복 뉴스와 시장 관련성이 낮은 뉴스는 삭제하지 않고 플래그로 표시한다.
- AI 뉴스 분석 기본 대상은 활성, 비중복, 분석 후보, 시장 관련성 기준을 통과한 뉴스만 사용한다.
- 개발용 force 실행을 제외하고 포토, 스포츠, 연예, 공지성 뉴스는 분석 후보에서 제외한다.
- 뉴스 품질 테스트에서 실제 Naver API를 호출하지 않는다.
- 뉴스 원문 전체나 외부 API raw 응답 전체를 Error Center context에 저장하지 않는다.

## 테마 분석 규칙

- 테마는 반드시 저장된 뉴스 분석 결과를 근거로 해야 한다.
- 입력에 없는 뉴스 ID와 회사명을 허용하지 않는다.
- 테마 순위는 GPT 출력만 믿지 않고 서버에서 계산한 점수로 정한다.
- 근거가 부족하면 테마 3개를 억지로 만들지 않는다.
- 종목 코드와 목표수익률은 이번 단계에서 생성하지 않는다.
- 실제 OpenAI 호출 테스트는 mock 처리한다.
## KRX 기준정보 규칙

- 승인되지 않은 KRX endpoint나 파라미터 이름을 추측해서 하드코딩하지 않는다.
- KRX API 키, 요청 헤더, 전체 응답 원문, 전체 종목 목록을 로그, 테스트, Error Center context에 저장하지 않는다.
- 빈 KRX 응답이나 부분 응답만으로 기존 한국 종목을 대량 비활성화하지 않는다.
- 한국 종목 코드는 6자리 문자열로 보존한다.
- 우선주, 스팩, ETF를 일반 보통주와 구분한다.
- 실제 KRX API 테스트는 mock과 fixture로 처리한다.

## 개발자 설정 규칙

- API 비밀값을 SQLite, 로그, 테스트 fixture, Error Center context에 저장하지 않는다.
- keyring 저장 실패 시 평문 파일 fallback을 만들지 않는다.
- 개발자 설정 API는 로컬 요청만 허용한다.
- 상태 API는 설정 여부와 출처만 반환하고 실제 값을 반환하지 않는다.
- 외부 연결 테스트는 최소 요청만 수행한다.
- 자동 테스트에서 실제 외부 API나 실제 Windows Credential Manager를 사용하지 않는다.
- Error Center 기록 전 비밀값을 마스킹한다.
## 테마 후보 생성 규칙

- GPT가 만든 ticker를 신뢰하지 않고 실제 종목 마스터 검증을 통과한 후보만 저장한다.
- 특정 기업 하나만 하드코딩해서 태그 문제를 해결하지 않고, AI 결과와 규칙 기반 태그 보강을 결합한다.
- 뉴스 직접 영향 산업과 언급 기업의 본업 산업을 구분해서 저장한다.
- 후보 검색 태그는 이슈, 직접 산업, 기업 본업 산업, 시장 테마, 동의어를 함께 사용한다.
- 부정적 테마는 기본적으로 후보 생성 대상이 아니라 위험 알림 또는 주의 테마로 다룬다.
- 회사명이 없는 테마에서 개별 주식을 과도하게 생성하지 않는다.
- ambiguous 후보는 자동 추천하지 않는다.
- 후보 점수는 뉴스 근거, 테마 점수, 종목 매칭 점수, 위험 패널티를 함께 사용한다.
- 현재가, 가격 전망, 수익 전망은 후보 생성 단계에서 만들지 않는다.

## AI 분석 UX 규칙

- AI 뉴스 분석, 단기 테마 분석, 테마 후보 생성은 통합 실행 API로 이어서 실행할 수 있어야 한다.
- 기존 개별 실행 API는 삭제하지 않는다.
- 분석 건수와 기간 같은 숫자 설정은 고정 selectbox보다 직접 입력 가능한 number_input을 우선한다.
- 통합 실행 응답은 단계별 결과를 `news_analysis`, `theme_analysis`, `candidate_generation`으로 분리해서 반환한다.
- 후보 생성 결과를 실제 투자 판단처럼 표시하지 않는다.
- 일반 사용자 화면은 `시장 분석` 중심으로 단순화하고, AI 분석/테마 분석의 단독 실행과 진단 기능은 `개발자 도구`의 관리 탭으로 둔다.
- 메인 메뉴에서 `AI 분석`을 별도 탭으로 노출하지 않는다.

## 최종 관심 후보 엔진 규칙

- 최종 관심 후보 엔진은 저장된 테마 후보 점수와 종목 마스터 검증 결과만 사용한다.
- GPT가 만든 ticker를 새 최종 후보로 직접 신뢰하지 않는다.
- 최종 주식 관심 후보의 기본 국가 범위는 `KR_ONLY`이며, 미국 주식은 기본적으로 해외 참고 후보로 분리한다.
- ambiguous, unmatched 후보는 자동 선정하지 않는다.
- 후보가 비어 있을 때 단순 빈 표로 표시하지 않고, no_kr_stock_candidate, only_us_candidates, ambiguous_only 같은 안전한 사유를 함께 보여준다.
- 뉴스 선택 기준과 실제 사용된 뉴스 요약은 사용자가 확인할 수 있어야 하며, 뉴스 원문 전체는 표시하거나 저장하지 않는다.
- 국내 주식, 국내 ETF, 해외 참고, 제외 후보, 매칭 불확실 후보를 UI에서 섞지 않는다.
- 실제 거래 지시, 확정 이익처럼 보이는 표현, 가격 전망, 수익 전망을 만들지 않는다.
- 가격 데이터가 없으면 가격·수익 관련 필드를 임의 생성하지 않는다.
- 레버리지/인버스 ETF는 기본 제외하고, 포함 옵션이 켜진 경우에도 위험 플래그를 표시한다.
